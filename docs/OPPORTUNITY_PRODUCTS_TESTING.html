Opportunity Products Testing

Sandbox Validation Checklist for Hybrid ACV Integration

SANDBOX TESTING PHASE

⚠️ Pre-Testing Requirements

Complete these steps before running test cases:

0 Prerequisites Checklist

1 Test: Opp Without Products or ACV

Default ACV Application

Steps

  1. Create new Opportunity
  2. Leave ACV field blank
  3. Do NOT add any Products
  4. Save Opportunity

Expected Result

Default ACV is applied by "Create Opp Assist v2" flow (existing behavior preserved)

2 Test: Opp With Products, No Manual ACV

Products Populate ACV

Steps

  1. Create new Opportunity, leave ACV blank, Save
  2. Click "Add Products" button
  3. Select "AI-Augmented Contracting - In-House Technology" ($120,000)
  4. Select "Sigma" ($120,000)
  5. Save Products
  6. Refresh Opportunity page

Expected Result

ACV = $240,000 (sum of line items)

Amount field also shows $240,000 (standard rollup)

3 Test: Opp With Manual ACV, Then Add Products

Manual ACV Preserved

Steps

  1. Create new Opportunity
  2. Enter ACV = $500,000
  3. Save Opportunity
  4. Click "Add Products"
  5. Select any product ($120,000)
  6. Save Products
  7. Refresh Opportunity page

Expected Result

ACV = $500,000 (manual entry preserved, NOT overwritten)

Amount = $120,000 (standard rollup, separate from ACV)

4 Test: Update Product Prices

Product Update Behavior

Steps

  1. Use Opportunity from Test 2 (ACV was set from Products)
  2. Edit one of the Products
  3. Change Sales Price from $120,000 to $150,000
  4. Save
  5. Refresh Opportunity page

Expected Result

ACV = $270,000 (updated sum: $150K + $120K)

Since ACV was originally set from Products (matched Amount), it updates with Products

5 Test: Delete Product

Product Deletion Behavior

Steps

  1. Use Opportunity from Test 4
  2. Delete one Product (e.g., the $120,000 one)
  3. Confirm deletion
  4. Refresh Opportunity page

Expected Result

ACV = $150,000 (only remaining product)

ACV recalculated because it matched Amount before deletion

6 Test: Stage Snapshot Capture

Stage Change Records ACV

Steps

  1. Create Opportunity with Products (ACV auto-populated)
  2. Change Stage from current to next stage
  3. Save
  4. Check Stage Snapshot related list

Expected Result

New Stage Snapshot record created with:

  • ACV = Current Opportunity ACV__c value
  • Stage = New stage name
  • Target Sign Date = Current Target_LOI_Date__c

7 Test: Delivery Creation (Stage 4)

Multi-Product Delivery Records

Steps

  1. Create Opportunity with 2+ Products
  2. Progress through stages to Stage 4 (Proposal)
  3. Save
  4. Check Delivery related list

Expected Result

Multiple Delivery records created — one per product

Each Delivery has:

  • Name: [Account] - [Product Name]
  • Product Line: Product name or family
  • ACV: Individual product value

Note: This test requires updated Delivery trigger (Phase 3)
